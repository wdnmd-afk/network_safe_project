# 网络钓鱼识别脚本目录

## 当前状态

当前目录只保留说明文档，不提供 `exploit.py` 或 `verify.ts`。

`social/phishing` 首版定位为 `case-study`，当前 planned 切片只登记文档入口，不登记 scripts 入口。

## 后续允许的脚本

后续最多允许新增只读一致性验证脚本：

- 读取 `labs/social/phishing/meta.json`。
- 检查文档入口是否存在。
- 检查固定案例 key、风险标签和安全边界是否一致。
- 检查不存在 `exploit.py`、发送能力、凭据收集能力或模板生成能力。

## 禁止能力

- 不发送真实邮件、短信、消息或链接。
- 不请求外部 URL。
- 不连接第三方邮件、短信、社交平台或收件箱服务。
- 不生成可投递邮件模板包、标题库、正文模板库或群发脚本。
- 不读取 `.env`、Cookie、token、验证码或凭据。
- 不提供网络钓鱼攻击脚本。
