# 捕鲸攻击脚本目录

## 当前状态

当前目录用于 `social/whaling` in-progress 阶段的脚本边界说明和本机只读一致性验证。

`social/whaling` 当前已建立标准目录、基础文档、docs 元数据入口、前端固定案例工作台、Playwright 页面差异验证、本机只读一致性验证和后端受控 review API。当前只登记 `verify.ts` 只读验证入口，不提供 `exploit.py`、投递脚本、画像采集脚本、模板生成脚本、付款指令脚本或第三方平台连接脚本。

## 当前脚本

- `verify.ts`：只读取仓库内捕鲸攻击元数据、文档、前端、后端和测试文件，输出 JSON 一致性报告。

运行方式：

```bash
pnpm --filter @network-safe/web exec tsx ../../tools/lab-scripts/social/whaling/verify.ts
```

`verify.ts` 不是攻击脚本，不发起 HTTP 请求，不发送邮件、短信、消息或会议邀请，不连接第三方平台、通讯录、CRM、HR、IM、日程、支付或收件箱服务。

## 禁止能力

- 不发送真实邮件、短信、消息、会议邀请或链接。
- 不请求外部 URL。
- 不连接第三方邮件、短信、社交平台、CRM、HR、IM、企业通讯录、会议系统、日程系统、投递服务、收件箱服务或支付系统。
- 不采集真实高管、董事会成员、外部顾问、供应商、客户、姓名、邮箱、手机号、公司、部门、职位、汇报关系、组织结构、社交账号或公开资料。
- 不生成可投递标题库、正文模板库、IM 话术库、会议邀请模板、附件诱导文案、付款指令、跟踪链接或群发脚本。
- 不读取 `.env`、Cookie、token、验证码、凭据、付款信息、法务材料、董事会材料、并购材料或真实业务资料。
- 不提供捕鲸攻击脚本。
- 不将 `verify.ts` 用作投递器、画像采集器、模板生成器、付款指令脚本或攻击脚本。

## 后续约束

后续如果进入 ready 收口，必须单独编写执行文档，并继续确认 `verify.ts` 只读取仓库内文件，不发起 HTTP 请求，不连接网络，不读取真实凭据或真实业务材料。
