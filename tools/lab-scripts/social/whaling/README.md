# 捕鲸攻击脚本目录

## 当前状态

当前目录仅用于 `social/whaling` planned 阶段的脚本边界说明。

`social/whaling` 当前只建立标准目录、基础文档和 docs 元数据入口。当前不登记 scripts 入口，不提供 `verify.ts`、`exploit.py`、投递脚本、画像采集脚本、模板生成脚本、付款指令脚本或第三方平台连接脚本。

## 禁止能力

- 不发送真实邮件、短信、消息、会议邀请或链接。
- 不请求外部 URL。
- 不连接第三方邮件、短信、社交平台、CRM、HR、IM、企业通讯录、会议系统、日程系统、投递服务、收件箱服务或支付系统。
- 不采集真实高管、董事会成员、外部顾问、供应商、客户、姓名、邮箱、手机号、公司、部门、职位、汇报关系、组织结构、社交账号或公开资料。
- 不生成可投递标题库、正文模板库、IM 话术库、会议邀请模板、附件诱导文案、付款指令、跟踪链接或群发脚本。
- 不读取 `.env`、Cookie、token、验证码、凭据、付款信息、法务材料、董事会材料、并购材料或真实业务资料。
- 不提供捕鲸攻击脚本。

## 后续约束

后续如果补充只读一致性验证脚本，必须单独编写执行文档，并且脚本只能读取仓库内元数据、文档和边界文件，不发起 HTTP 请求，不连接网络，不读取真实凭据或真实业务材料。
