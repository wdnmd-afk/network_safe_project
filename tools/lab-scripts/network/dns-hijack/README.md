# DNS 劫持 / 污染脚本目录

## 1. 当前状态

当前目录用于记录脚本边界，并提供一个本机只读一致性验证脚本。

该脚本用于校验 `network.dns-hijack` 元数据、文档、前端入口、后端 API、页面级验证证据和安全边界是否一致，但不是攻击脚本，也不是 DNS 查询脚本。

本阶段不创建：

- `exploit.py`
- 真实 DNS 查询脚本
- 系统网络配置脚本

当前可执行脚本：

- `verify.ts`：只读取本仓库内 DNS 劫持元数据、文档和实现文件，确认入口、自动化证据和安全边界一致。

当前不提供 exploit.py、真实 DNS 查询脚本或系统网络配置脚本。

## 2. 后续脚本边界

如果后续需要新增脚本，只允许：

- 读取本仓库内固定元数据和文档。
- 输出本机受控学习信号。
- 在明确执行文档约束下调用本项目 localhost API。

禁止：

- 接收任意域名、DNS 服务器、IP、代理、网络接口或端口参数。
- 请求真实外部 DNS、DoH、DoT 或公共解析服务。
- 修改本机 DNS、hosts、代理、路由、防火墙或系统网络配置。
- 实现真实 DNS 投毒、DNS 劫持、DNS 隧道、流量转发或中间人代理。
- 输出可复用到外部目标的真实解析或劫持结果。
