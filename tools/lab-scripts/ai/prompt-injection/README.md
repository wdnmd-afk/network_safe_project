# Prompt 注入脚本目录

当前目录用于 `ai/prompt-injection` ready 阶段的脚本边界说明和本机只读一致性验证。

当前已有前端固定样例工作台、后端确定性路由 API、统一事件日志安全摘要、前端单元测试、Playwright 页面验证、服务端 API 测试和本机只读一致性验证。

当前唯一脚本入口：

- `verify.ts`：只读取仓库内元数据、文档、前端工作台、前端 API client、前端模型、后端服务实现和 Playwright 验证文件，输出一致性验证报告。

当前不提供：

- `exploit.py`
- 外部 AI 调用脚本
- 模型测试脚本
- 任意提示词执行器
- 可复用绕过模板

`verify.ts` 不发起 HTTP 请求，不调用外部 AI、真实工具、第三方 API 或模型服务，不读取 `.env`、Cookie、token 或凭据，也不接收任意提示词、URL、模型名、API key、工具参数或外部目标。

后续如需新增其他脚本，必须单独编写执行文档，并且不得突破当前安全边界。
