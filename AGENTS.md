# 项目级协作规则

本文件用于约束 `network_safe_project` 仓库内后续所有开发、文档、脚本与测试工作。若与外层全局规则同时存在，默认在不冲突前提下同时生效；若项目内实现细节需要收紧，以本文件为准。

## 1. 项目定位

本项目是一个运行于 Windows 本机环境的网络安全学习平台，核心目标如下：

- 个人学习优先
- 尽量覆盖常见网络安全问题
- 同一场景尽量同时提供漏洞版与修复版
- 为后续自动化测试预留统一结构

本项目不是通用攻击工具库，不以对外攻击能力为目标。

## 2. 基础技术约束

- 前端技术栈：Vue
- 后端技术栈：Node.js
- 数据库：MySQL
- 运行环境：Windows 本机
- 部署方式：前端可由 `nginx` 托管，后端 Node 服务独立运行
- 暂不使用 Docker
- 仓库结构采用 monorepo

## 3. 目录约束

后续实现应优先遵循以下目录分层：

- `apps/`：前后端应用
- `packages/`：共享类型、场景规范、内容索引
- `labs/`：按场景组织的实验内容
- `tools/lab-scripts/`：按场景组织的实验脚本与验证资源
- `database/`：数据库结构、种子与场景数据
- `docs/`：设计、执行、测试、场景文档
- `nginx/`：本机部署配置

禁止将实验脚本、样本文件、自动化验证文件随意散落在业务目录中。

## 4. 场景实现规则

每个场景应尽量使用统一结构：

```text
labs/<category>/<scene>/
├─ meta.json
├─ README.md
├─ vuln/
├─ fixed/
├─ mock/
└─ docs/
```

要求：

- `vuln/` 与 `fixed/` 必须职责清晰，不得混写成不可区分的同层实现
- `meta.json` 作为统一注册入口
- `README.md` 必须说明场景目标、前置条件、使用方式
- `docs/` 用于记录攻击步骤、修复思路、注意事项

## 5. 脚本目录规则

实验脚本目录固定为：

```text
tools/lab-scripts/<category>/<scene>/
```

推荐内容：

- `README.md`
- `exploit.py`
- `verify.ts`
- `samples/`
- `artifacts/`

约束如下：

- 脚本仅服务于本项目中的受控学习场景
- 优先按场景组织，不按语言优先组织
- Python 用于实验与验证脚本
- TypeScript 用于与平台集成的验证逻辑和自动化测试辅助
- 不得将该目录扩展为通用攻击工具仓库

## 6. 字段与接口规则

- 禁止猜测字段名
- 禁止写“多字段碰运气兜底”逻辑
- 场景元数据字段、接口字段、数据库字段必须先确认来源
- 字段来源优先级：
  1. 类型定义
  2. 数据库结构
  3. 场景元数据规范
  4. 已有实现
  5. 明确文档

若字段不明确，先补文档或确认结构，再实现。

## 7. 文档规则

- 关键文档默认使用中文
- 复杂任务必须先写执行文档
- 文档目录默认遵循：
  - `docs/execution/`
  - `docs/design/`
  - `docs/labs/`
  - `docs/testing/`
- 方案、阶段拆分、风险分析、验证方式必须可追溯

## 8. 测试与验证规则

- 未完成验证前，不得声称已完成实现
- 默认优先做最小必要验证
- 非用户明确要求，不执行高成本全量构建
- 后续自动化测试按三层建设：
  - 单元 / 模块测试
  - API / 集成测试
  - 端到端测试

每个场景后续尽量具备：

- 漏洞版验证方式
- 修复版验证方式
- 对应脚本或测试入口

## 9. 安全边界规则

本项目允许研究和展示常见攻击方式，但必须保持以下边界：

- 只面向本机、本项目、受控实验环境
- 不将脚本描述为对外部真实目标的攻击工具
- 对不适合真实复现的攻击类型，优先采用模拟演示、案例讲解、脚本实验等方式
- 不为了“全覆盖”牺牲结构清晰度与学习价值

## 10. 实施优先级

后续建设默认按以下顺序推进：

1. monorepo 基础骨架
2. 平台核心目录与共享规范
3. 场景元数据结构
4. 数据库基础表
5. 一期 Web / 认证授权场景
6. 脚本目录与自动化验证入口
7. 网络实验与社会工程学模块

## 11. 提交与修改原则

- 优先遵循现有结构与命名
- 非必要不引入新依赖
- 非必要不做无关重构
- 修改前先阅读上下游
- 涉及跨模块改动时，先补文档再实施

## 12. 当前阶段说明

当前仓库处于规划期。

在进入代码实现前，至少应先完成以下文档或结构确认：

- 根目录 `README.md`
- 项目级 `AGENTS.md`
- 执行文档
- 目录骨架
- 场景元数据规范
- 一期场景清单
