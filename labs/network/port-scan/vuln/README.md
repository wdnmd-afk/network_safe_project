# 漏洞版观察

## 1. 定位

漏洞版用于展示虚拟资产暴露多个不必要端口后的风险。

漏洞版页面入口为 `/labs/network/port-scan/vuln`，页面只调用本项目受控虚拟资产 API。

## 2. 当前观察点

当前漏洞版受控 API 使用固定虚拟目标展示：

- 管理端口可见。
- 调试端口可见。
- 多余服务增加攻击面。
- 开放端口数量和风险端口数量偏高。

## 3. 安全边界

漏洞版仍只使用虚拟资产表，不做真实端口扫描。

禁止：

- 提供 `exploit.py` 或真实扫描脚本。
- 输入任意目标。
- 扫描真实网络。
- 调用系统探测命令。
- 输出真实服务指纹。

## 4. 当前学习信号

- `port-scan-exposure-expanded`
- `port-scan-management-surface-visible`
