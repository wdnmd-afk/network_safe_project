# 攻击方观察步骤

## 1. 当前阶段

当前实验处于 `in-progress` 阶段，已提供后端受控 API，尚未提供页面或脚本入口。

本文件只描述后续学习路径，不提供真实扫描命令。

## 2. 当前 API 观察步骤

当前可通过本项目后端 API 观察固定虚拟目标：

1. 登录本项目演示账号，获取本机 session。
2. 选择一个固定虚拟目标，例如 `admin-node`。
3. 选择固定观察模式，例如 `surface-review`。
4. 提交到 `POST /api/labs/network/port-scan/vuln/scan`。
4. 查看虚拟开放端口列表和暴露面评分。
5. 观察管理端口、调试端口和多余服务带来的学习信号。
6. 对照修复版查看暴露面收敛后的差异。

## 3. 攻击方视角

本实验中的攻击方视角只用于理解侦察风险：

- 开放端口越多，攻击者可推断的信息越多。
- 管理端口可见会提高后续攻击链风险。
- 多余服务会增加补丁、配置和访问控制负担。

## 4. 明确禁止

本实验不提供：

- 真实端口扫描命令。
- 任意目标输入。
- 网段扫描方式。
- 并发扫描方式。
- 服务漏洞识别或利用步骤。
