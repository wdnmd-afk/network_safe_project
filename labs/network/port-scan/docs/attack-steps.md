# 攻击方观察步骤

## 1. 当前阶段

当前实验处于 `in-progress` 阶段，已提供前端工作台和后端受控 API，尚未提供脚本入口。

本文件只描述受控学习路径，不提供真实扫描命令。

## 2. 当前页面观察步骤

当前可通过本项目前端页面观察固定虚拟目标：

1. 登录本项目演示账号。
2. 进入 `/labs/network/port-scan/vuln`。
3. 选择一个固定虚拟目标，例如 `admin-node`。
4. 选择固定观察模式，例如 `surface-review`。
5. 点击“观察暴露面”。
6. 查看虚拟开放端口列表、暴露面评分和学习信号。
7. 切换到 `/labs/network/port-scan/fixed`，使用同一目标观察暴露面收敛后的差异。

## 3. 当前 API 观察步骤

当前也可通过本项目后端 API 观察固定虚拟目标：

1. 登录本项目演示账号，获取本机 session。
2. 选择固定虚拟目标和固定观察模式。
3. 提交到 `POST /api/labs/network/port-scan/vuln/scan` 或 `POST /api/labs/network/port-scan/fixed/scan`。
4. 只检查返回的虚拟端口、虚拟统计和学习信号。

## 4. 攻击方视角

本实验中的攻击方视角只用于理解侦察风险：

- 开放端口越多，攻击者可推断的信息越多。
- 管理端口可见会提高后续攻击链风险。
- 多余服务会增加补丁、配置和访问控制负担。

## 5. 明确禁止

本实验不提供：

- 真实端口扫描命令。
- 任意目标输入。
- 网段扫描方式。
- 并发扫描方式。
- 服务漏洞识别或利用步骤。
