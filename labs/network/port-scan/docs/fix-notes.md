# 修复思路

## 1. 修复目标

端口暴露面治理的目标不是“隐藏一切”，而是让必要服务清晰、最小、可审计。

## 2. 核心策略

当前修复版前端工作台和受控 API 展示：

- 只开放必要业务端口。
- 管理端口只允许受控网络访问。
- 调试端口默认关闭。
- 下线无业务价值的服务。
- 维护资产清单和端口责任人。
- 对异常暴露面变化进行审计。

## 3. 生产环境补充

真实生产环境中还需要：

- 防火墙策略。
- 网络分区。
- 零信任访问控制。
- 持续资产发现。
- 变更审批。
- 日志和告警联动。

## 4. 本项目边界

本项目只做虚拟资产演示，不修改本机防火墙、路由、DNS、hosts、代理或真实服务配置。

当前只读一致性验证脚本也只检查仓库内文档、元数据和实现边界，不调用真实网络探测能力。
