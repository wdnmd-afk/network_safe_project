# 修复思路

## 1. 修复目标

DNS 劫持 / 污染防护的目标不是让解析过程变成黑盒，而是让解析来源可信、解析结果可校验、异常解析可审计。

## 2. 核心策略

当前修复版 API 已展示：

- 使用可信解析源。
- 校验解析结果是否落在期望虚拟地址类别。
- 校验证书状态是否匹配。
- 对异常解析结果进行阻断或审计。
- 将异常解析行为写入统一事件日志。

## 3. 生产环境补充

真实生产环境中还需要：

- 终端 DNS 配置治理。
- 递归解析器安全加固。
- 域名和证书监控。
- HSTS、证书透明度和供应商审计。
- 网络侧异常流量监控。
- 用户侧风险提示和应急流程。

## 4. 本项目边界

本项目只做内存解析表模拟，不修改本机 DNS、hosts、代理、路由、防火墙或真实服务配置。

当前后端 API 只调用内存解析表。当前只读一致性验证脚本只读取仓库文件；后续如新增脚本，也只能在新的执行文档约束下读取仓库文件或调用本项目 localhost API，不请求外部 DNS。
