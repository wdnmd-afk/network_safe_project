# DNS 劫持 / 污染修复版说明

## 1. 当前状态

当前为 `ready` 阶段，已实现修复版页面 `/labs/network/dns-hijack/fixed`、修复版 `POST /api/labs/network/dns-hijack/fixed/resolve`、页面级验证和本机只读一致性验证。

## 2. 修复版目标

修复版用于展示同一固定域名样例在可信解析源、证书校验和异常审计下的防御效果。

建议观察点：

- 解析结果是否恢复到期望虚拟地址类别。
- 证书状态是否保持 `trusted`。
- 异常解析是否被阻断或记录审计信号。
- 后端学习信号是否为 `dns-hijack-trusted-resolution-restored` 或 `dns-hijack-anomaly-blocked`。
- 统一事件日志是否只记录固定样例摘要，不保存真实域名、真实 IP 或真实 DNS 响应。
- 前端页面是否只提交固定 `domainKey` 和固定 `resolverProfile`。

## 3. 防御方视角

修复版重点不是让解析过程不可见，而是让解析结果可信、可校验、可审计。

后续修复策略建议包含：

- 固定可信解析源。
- 校验解析结果类别。
- 校验证书状态。
- 记录异常解析审计信号。
- 保持正常域名样例可用。

## 4. 本项目边界

本项目只做内存解析表模拟，不修改本机 DNS、hosts、代理、路由、防火墙或真实网络配置。
