# 中间人攻击

## 场景目标

使用固定握手记录模拟链路身份校验缺失与证书验证差异。

本实验只使用固定案例 `synthetic-handshake-interception`，用于观察客户端未验证证书链、主机名或协议降级，并对比漏洞版与修复版的后端判定、学习信号和统一事件日志。

## 前置条件

- 本机前端与后端服务已启动。
- 使用本项目本机演示账号登录。
- 不需要外部目标、第三方服务、真实凭据或真实业务数据。

## 使用方式

1. 访问 `/labs/network/mitm/vuln`。
2. 使用固定案例和默认控制策略观察风险被接受。
3. 切换到 `/labs/network/mitm/fixed`，观察高风险动作被阻断。
4. 选择“证书身份已验证”，确认正常受控流程仍可继续。
5. 在实验详情或账户中心复盘统一事件日志。

## 安全边界

- 只使用内存中的虚拟网络拓扑和固定流量摘要，不扫描、监听或连接真实网络。
- 页面和 API 只接受共享目录中声明的 scenarioKey 与 controlKey。
- 未知 key 会被脱敏阻断，不写入原始输入或外部目标信息。

该实验仅提供本机受控固定场景和只读验证，不允许扩展为通用攻击工具。
