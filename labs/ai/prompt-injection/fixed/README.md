# Prompt 注入修复版规划

## 1. 学习目标

修复版用于从防御方视角观察：指令分层、检索隔离、工具允许列表和输出策略校验如何阻止外部内容覆盖业务意图。

当前已提供前端固定样例工作台和后端确定性路由 API，不提供防御验证脚本。

## 2. 当前防御点

修复版工作台和 API 使用固定样例展示：

- 系统意图与外部内容分层。
- 检索片段只作为资料，不作为高优先级指令。
- 工具请求只允许固定工具标签，不执行真实工具调用。
- 输出策略校验将越界请求转成安全解释或阻断结果。
- 事件日志只保存固定 key、输入摘要长度、风险类别、策略状态和学习信号。

当前 API：

```text
POST /api/labs/ai/prompt-injection/fixed/evaluate
```

当前页面：

```text
/labs/ai/prompt-injection/fixed
```

请求只读取：

- `scenarioKey`
- `instructionSourceKey`
- `defensePolicyKey`

## 3. 安全边界

修复版不得：

- 依赖前端隐藏输入框作为主要修复。
- 调用真实模型或真实工具。
- 记录完整提示词、完整检索片段、真实模型输出、Cookie、token 或凭据。
- 展示可复制危险提示词、绕过模板、钓鱼文本、恶意代码或仿冒身份内容。

## 4. 后续完成条件

后续只有在单独执行文档明确后，才允许新增前端自动化测试或只读验证脚本。当前 `in-progress` 阶段只表示修复版前端工作台、后端 API、文档和元数据入口已建立。
