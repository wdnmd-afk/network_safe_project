# Prompt 注入修复版规划

## 1. 学习目标

修复版用于从防御方视角观察：指令分层、检索隔离、工具允许列表和输出策略校验如何阻止外部内容覆盖业务意图。

本阶段只是规划文档，不提供可运行页面、API 或防御验证脚本。

## 2. 后续防御点

后续修复版应使用固定样例展示：

- 系统意图与外部内容分层。
- 检索片段只作为资料，不作为高优先级指令。
- 工具请求只允许固定工具标签，不执行真实工具调用。
- 输出策略校验将越界请求转成安全解释或阻断结果。
- 事件日志只保存长度、类别、样例 key 和学习信号。

## 3. 安全边界

修复版不得：

- 依赖前端隐藏输入框作为主要修复。
- 调用真实模型或真实工具。
- 记录完整提示词、完整检索片段、真实模型输出、Cookie、token 或凭据。
- 展示可复制危险提示词、绕过模板、钓鱼文本、恶意代码或仿冒身份内容。

## 4. 后续完成条件

后续只有在单独执行文档明确后，才允许新增受控 API、前端工作台、自动化测试或只读验证脚本。当前 planned 阶段只表示修复版说明和元数据入口已建立。
