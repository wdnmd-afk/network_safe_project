# Prompt 注入修复说明

## 1. 当前状态

当前 `ai/prompt-injection` 处于 `in-progress` 状态，修复版已有后端确定性路由 API 和服务端 API 测试，没有前端页面或只读验证脚本。

## 2. 当前修复策略

当前修复版 API 覆盖以下策略：

- 指令分层：系统意图、开发约束、检索内容和用户请求必须有清晰优先级。
- 检索隔离：外部资料只能作为参考内容，不允许改变系统意图。
- 工具允许列表：只允许固定教学工具标签，不执行真实工具调用。
- 输出策略校验：越界请求应被阻断或转成安全化教学回答。
- 日志审计：只记录场景 key、来源 key、防御策略 key、输入长度、风险类别和学习信号。

当前 API：

```text
POST /api/labs/ai/prompt-injection/fixed/evaluate
```

修复版对固定风险样例返回阻断结果，并对固定正常文档问答样例返回安全教学回答。

## 3. 真实生产补充

真实生产中还需要结合：

- 真实模型供应商安全策略。
- 权限隔离和工具调用审计。
- 检索内容来源可信度评估。
- 敏感数据最小化。
- 异常输出监控和人工复核流程。

本项目首版不实现这些真实生产集成，只做本机受控学习模拟。

## 4. 禁止行为

修复说明不得建议：

- 调用外部 AI 生成攻击内容。
- 保存完整提示词或真实模型输出。
- 通过真实攻击样例测试第三方模型。
- 输出可复制绕过模板、钓鱼文本、恶意代码或仿冒身份内容。
