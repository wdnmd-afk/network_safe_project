# Prompt 注入漏洞版规划

## 1. 学习目标

漏洞版用于从攻击方视角观察：当外部内容、检索片段或用户输入被错误提升为高优先级指令时，业务目标如何偏离。

本阶段只是规划文档，不提供可运行页面、API 或攻击脚本。

## 2. 后续观察点

后续漏洞版应使用固定样例展示：

- 指令来源混淆。
- 检索内容未隔离。
- 工具请求越界暴露。
- 输出策略缺失。
- 日志只记录风险类别和学习信号。

后续学习者应观察固定样例的路由结果，而不是输入任意提示词。

## 3. 安全边界

漏洞版不得：

- 提供完整危险提示词。
- 提供可复制绕过模板。
- 生成钓鱼文本、恶意代码或仿冒身份内容。
- 调用真实 AI、真实工具或第三方接口。
- 接收外部目标、URL、密钥、模型配置或系统提示词。
- 保存完整用户提示词、完整系统提示词或完整检索片段。

## 4. 后续完成条件

后续只有在单独执行文档明确后，才允许新增后端受控 API 或前端工作台。当前 planned 阶段只表示漏洞版说明和元数据入口已建立。
