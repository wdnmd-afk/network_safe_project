# 高权威误判观察版

## 目标

本目录用于记录捕鲸攻击漏洞版的 case-study ready 说明。当前只能通过前端固定案例工作台、固定虚构线索卡和后端受控 review API 观察高权威、保密理由、重大业务事件和时间压力如何诱发越级审批、可信通道缺失或例外流程失守。

## 当前状态

当前已实现固定案例页面 `/labs/social/whaling/vuln`，后端已提供 `POST /api/labs/social/whaling/vuln/review`。页面和 API 只读取固定 `caseKey` 和 `verificationPolicyKey`，并写入统一事件日志安全摘要；当前仅提供本机只读一致性验证脚本 `verify.ts`，不提供攻击脚本能力。

## 允许观察的内容

- 高层授权压力如何让接收者过度相信请求。
- 董事会、法务、外部顾问或并购语境如何制造保密压力。
- 大额付款、临时资料室访问和例外审批如何压缩核验时间。
- 缺少可信通道回拨、双人复核或例外冻结时会出现哪些学习信号。

## 禁止内容

- 不写完整邮件正文、IM 对话、会议邀请、可复制标题或付款指令。
- 不采集真实高管、组织结构、汇报关系、公开资料、邮箱、手机号或职位链路。
- 不生成投递模板、附件诱导文案、跟踪链接、群发脚本或攻击脚本。
- 不提供 `exploit.py` 或任何可面向第三方目标使用的能力。
