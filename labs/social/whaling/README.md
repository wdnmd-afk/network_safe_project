# 捕鲸攻击

## 场景目标

本实验用于学习捕鲸攻击中更高风险的流程误判：高层授权压力、重大业务事件、董事会或外部顾问语境、保密理由、时间窗口压缩、大额付款、临时资料室访问和越级审批如何造成流程失守。

当前状态为 `case-study` / `planned`。planned 仅表示本项目内标准目录、元数据和基础文档入口已建立，不表示页面、API、数据库写入、事件日志写入、自动化验证或脚本能力已经实现。

## 当前范围

- 已建立 `social.whaling` planned 元数据。
- 已建立漏洞版 / 修复版说明目录。
- 已建立攻击方观察、修复说明和手动验证文档。
- 已建立固定虚构案例卡文档。
- 已建立脚本目录边界说明。
- 元数据当前只登记 docs 入口。

当前未实现：

- 前端页面或路由。
- 后端 API、service 或事件日志写入。
- `verify.ts`。
- `exploit.py`。
- 任何真实投递、高管画像采集、组织结构收集、凭据收集、模板生成、会议邀请、付款指令或第三方平台调用能力。

## 固定案例方向

当前已在 `docs/fixed-cases.md` 固化四个虚构案例方向：

- `executive-wire-approval`：观察高层口吻与紧急大额付款造成的审批压力。
- `board-confidential-request`：观察董事会保密事项如何让普通流程被绕过。
- `legal-settlement-transfer`：观察法务和外部顾问语境制造的紧急合规压力。
- `ma-data-room-access`：观察并购或投融资事件中的临时资料室访问诱导。

这些案例只作为固定线索卡、流程节点、风险标签和复盘问题使用，不提供完整邮件正文、IM 对话、可复制标题、会议邀请模板、可投递附件名、真实链接或付款指令。

## 固定学习信号

## 后续扩展边界

后续若进入后端固定案例 API、前端固定案例工作台、自动化验证或 ready 收口，必须单独编写执行文档。

后续 API 只能读取固定 `caseKey` 和固定 `verificationPolicyKey`。未知 key 必须阻断且不得回显原始输入。事件日志只能记录固定 key、风险类别、建议动作、后端决策和学习信号，不保存真实人员、邮箱、手机号、组织结构、链接、附件、凭据、付款信息、法务材料或正文。

## 安全边界

- 不发送真实邮件、短信、消息、会议邀请或链接。
- 不采集真实高管、董事会成员、外部顾问、供应商、客户、姓名、邮箱、手机号、公司、部门、职位、汇报关系、组织结构、社交账号或公开资料。

- 不连接第三方邮件、短信、社交平台、CRM、HR、IM、企业通讯录、会议系统、日程系统、投递服务、收件箱服务或支付系统。
- 不生成可投递标题库、正文模板库、IM 话术库、会议邀请模板、附件诱导文案、付款指令、跟踪链接或群发脚本。
- 不提供 `exploit.py`、`verify.ts`、投递脚本、画像采集脚本、模板生成脚本、付款指令脚本或攻击脚本。
