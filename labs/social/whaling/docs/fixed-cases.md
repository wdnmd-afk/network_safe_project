# 捕鲸攻击固定案例卡

## 1. 文档定位

本文档定义 `social/whaling` 后续学习入口可使用的固定虚构案例卡。

这些案例只用于观察高层授权压力、重大业务事件、董事会或外部顾问语境、保密理由、时间窗口压缩、大额付款、临时资料室访问和越级审批如何造成流程失守。本文档不提供完整邮件正文、IM 对话、会议邀请模板、可复制标题、可投递附件名、真实链接、付款指令、跟踪链接、模板库、群发脚本或第三方平台调用方式。

当前实验状态为 `case-study` / `planned`。本文档只是固定案例来源，不是页面、API、脚本、投递能力或自动化攻击能力。

## 3. 案例：`executive-wire-approval`

### 3.1 案例目标

学习者观察“高层授权标签 + 紧急大额付款语境”如何让执行者跳过正式付款核验。

### 3.2 虚构角色标签

- 请求方标签：高层决策角色。
- 执行动作标签：大额付款执行者。
- 流程标签：财务付款审批链。
- 核验标签：可信通道回拨和双人复核。

### 3.3 误判线索

- 高权威压力：请求看起来来自高层决策角色，接收者倾向于降低质疑。
- 紧急时限：请求强调窗口期，压缩常规核验。
- 审批跳过：请求暗示先执行、后补流程。
- 保密理由：请求使用保密语境弱化协作复核。

### 3.4 防御动作

- 检查付款是否经过正式财务审批链。
- 使用预先登记的可信通道回拨确认。
- 对大额付款和紧急例外启用双人复核。
- 在核验前冻结付款动作，并记录学习复盘日志摘要。

### 3.5 学习信号

- `whaling-executive-authority-overweighted`
- `whaling-confidential-pressure-identified`
- `whaling-payment-freeze-required`

## 4. 案例：`board-confidential-request`

### 4.1 案例目标

学习者观察“董事会保密事项 + 越级请求”如何让普通流程被绕过。

### 4.2 虚构角色标签

- 请求方标签：董事会事务角色。
- 执行动作标签：高敏事项协作执行者。
- 流程标签：董事会固定事务通道。
- 核验标签：法务核验和例外审批记录。

### 4.3 误判线索

- 保密压力：请求强调最小知情范围，削弱常规复核意愿。
- 越级请求：请求绕过固定事务通道，直接要求执行动作。
- 角色模糊：请求方标签带有高层或董事会语境，但缺少固定流程依据。
- 审计缺失：请求没有形成可追踪的例外审批记录。

### 4.4 防御动作

- 只通过董事会事务固定通道处理保密事项。
- 对例外请求记录审批链和复核人。
- 通过法务或合规负责人核验事项边界。
- 将不符合固定通道的请求隔离并进入复盘。

### 4.5 学习信号

- `whaling-confidential-pressure-identified`
- `whaling-payment-freeze-required`
- `whaling-boundary-verified`

## 5. 案例：`legal-settlement-transfer`

### 5.1 案例目标

学习者观察“法务事项 + 外部顾问语境 + 紧急付款”如何制造合规压力并诱发付款例外。

### 5.2 虚构角色标签

- 请求方标签：外部顾问协作角色。
- 执行动作标签：法务或财务协作执行者。
- 流程标签：案件编号和结算审批链。
- 核验标签：顾问白名单、法务负责人确认和财务复核。

### 5.3 误判线索

- 外部顾问权威：请求看似来自专业顾问或合规协作链路。
- 案件压力：请求强调处理窗口或结算期限。
- 付款例外：请求要求绕过常规付款或供应商流程。
- 信息不完整：请求缺少可核验案件编号、合同流程或正式审批记录。

### 5.4 防御动作

- 使用外部顾问白名单和固定联系渠道核验。
- 校验案件编号、合同流程和法务负责人确认。
- 将结算付款绑定财务审批链和双人复核。
- 未完成核验前冻结付款并记录风险类别。

### 5.5 学习信号

- `whaling-confidential-pressure-identified`
- `whaling-payment-freeze-required`
- `whaling-boundary-verified`

## 6. 案例：`ma-data-room-access`

### 6.1 案例目标

学习者观察“并购或投融资资料室 + 临时访问请求”如何让最小授权、有效期和独立审批被忽略。

### 6.2 虚构角色标签

- 请求方标签：重大交易协作角色。
- 执行动作标签：资料室权限管理员。
- 流程标签：临时资料室访问审批。
- 核验标签：独立审批、安全团队复核和权限有效期。

### 6.3 误判线索

- 重大交易语境：请求暗示业务重要性，压缩安全复核。
- 临时权限：请求要求扩大访问范围或延长有效期。
- 审批缺失：请求缺少独立审批记录。
- 最小授权失守：请求范围超过当前协作所需。

### 6.4 防御动作

- 要求资料室访问绑定独立审批和范围说明。
- 限制访问范围、有效期和可下载能力。
- 对高敏资料访问增加安全团队复核。
- 复盘权限是否按期回收，并只记录安全摘要。

### 6.5 学习信号

- `whaling-executive-authority-overweighted`
- `whaling-payment-freeze-required`
- `whaling-boundary-verified`
