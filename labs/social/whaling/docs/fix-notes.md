# 捕鲸攻击修复说明

## 修复目标

捕鲸攻击的防御重点不是识别某个单一字段，而是让高风险业务流程不能只依赖显示身份、职位、语气、保密要求或紧急程度。

## 核心防御策略

- 大额付款双人复核：付款、供应商变更和紧急转账必须经过两名授权人员确认。
- 可信通道回拨：高层请求、外部顾问请求和重大交易请求必须通过预先登记的可信通道确认。
- 法务核验：法律和外部顾问事项必须绑定案件编号、合同流程或法务负责人确认。
- 董事会固定通道：董事会保密事项必须走固定事务通道和例外审批记录。
- 例外冻结：高风险例外先冻结，再由财务、法务、安全或管理层复核。
- 最小授权：临时资料室访问必须有范围、有效期、审批记录和自动回收。
- 隔离举报：可疑链接、附件、会议邀请或请求先隔离，再通过安全流程复盘。
- 日志复盘：事件日志只保存固定 key、风险类别、决策、学习信号和建议动作。

## 后续实现约束

后续如果实现 API 或页面，只能使用固定虚构案例和固定核验策略，不允许用户输入真实高管、真实组织结构、真实付款信息、真实业务材料、真实链接、真实附件或真实凭据。

未知固定 key 必须阻断，并且不得回显原始输入。事件日志必须保持安全摘要，不保存原始正文、付款信息、法务材料、董事会材料或并购资料。
