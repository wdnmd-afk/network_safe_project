# 网络钓鱼识别误判观察版

## 定位

漏洞版不是网络钓鱼发送器，而是“容易误判的收件箱观察模式”。

当前页面只展示固定案例线索卡，并突出攻击者常利用的表面信号，例如显示名、紧急主题、附件提示或品牌外观。学习者从攻击方视角理解这些线索如何造成误判，但不会获得可投递素材。

## 预期观察

- 只看显示名时，可能忽略域名不一致。
- 只看紧急语气时，可能忽略凭据请求风险。
- 只看业务名称时，可能忽略附件诱导或上下文异常。
- 漏洞版后续可返回 `accepted` 一类固定判定，用于观察错误放行倾向。

## 当前状态

当前已按 case-study ready 标准完成固定案例学习闭环：

- 前端入口：`/labs/social/phishing/vuln`。
- `POST /api/labs/social/phishing/vuln/review` 只读取固定 `caseKey`、`reviewModeKey` 和 `defenseChecklistKey`。
- 统一事件日志只记录固定 key、风险标签数量、建议动作和学习信号。
- 服务端 API 测试和只读一致性验证脚本用于证明误判观察版与修复版差异。
- 没有 `exploit.py`、投递脚本、模板生成器或攻击脚本。

## 禁止能力

- 不展示完整可投递邮件正文。
- 不提供真实登录页、真实链接、真实附件或真实品牌素材。
- 不提供邮件标题库、正文模板库、群发脚本或投递参数。
- 不收集邮箱、凭据、验证码、Cookie 或 token。
