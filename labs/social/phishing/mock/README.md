# 网络钓鱼识别固定案例说明

## 目录用途

本目录用于后续记录固定案例 key、线索摘要和风险标签。当前只保留说明文档，不存放可投递邮件模板、真实附件或真实链接。

## 固定案例草案

### `invoice-urgent-review`

- 场景：发票复核提醒。
- 线索：紧急语气、附件诱导、业务上下文异常。
- 学习重点：不要只根据主题紧急程度决定是否打开附件。

### `account-security-alert`

- 场景：账号安全提醒。
- 线索：相似域名、凭据请求、二次确认缺失。
- 学习重点：需要核对真实服务入口和安全通道。

### `hr-benefit-update`

- 场景：人事福利更新。
- 线索：品牌仿冒、附件诱导、跨部门请求异常。
- 学习重点：需要通过内部确认流程核对来源。

## 边界

- 只存固定案例 key、线索摘要和风险标签。
- 不存真实邮箱地址、真实链接、真实附件、二维码或凭据。
- 不存完整邮件正文、标题库或正文模板库。
