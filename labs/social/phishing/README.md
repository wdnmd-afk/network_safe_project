# 网络钓鱼识别

## 场景目标

本实验用于学习网络钓鱼中常见的识别线索：相似域名、紧急语气、凭据请求、附件诱导、品牌仿冒和异常业务上下文。

当前状态为 `planned`，只建立文档和元数据入口。后续目标是做一个固定案例的仿真收件箱识别训练，而不是邮件发送器或模板生成器。

## 当前范围

- 已建立 `social.phishing` 元数据。
- 已建立漏洞版 / 修复版说明目录。
- 已建立攻击方观察、修复说明和手动验证文档。
- 当前未实现前端页面、后端 API、数据库日志写入或自动化脚本。

## 固定案例方向

后续只允许固定案例 key：

- `invoice-urgent-review`：观察发票复核场景中的紧急语气、附件诱导和业务上下文异常。
- `account-security-alert`：观察账号安全提醒场景中的相似域名、凭据请求和二次确认缺失。
- `hr-benefit-update`：观察人事福利更新场景中的品牌仿冒、附件诱导和确认通道风险。

这些案例只作为线索卡和风险标签使用，不提供完整可投递邮件正文。

## 安全边界

- 不发送真实邮件、短信、消息或链接。
- 不收集真实邮箱、真实凭据、验证码、Cookie 或 token。
- 不连接第三方邮件、短信、社交平台、投递服务或收件箱服务。
- 不生成可投递邮件模板包、标题库、正文模板库、群发脚本或攻击脚本。
- 不提供 `exploit.py`。
