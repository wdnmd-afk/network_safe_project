# 恶意包注入修复说明

## 根因

构建流程自动信任依赖更新且没有来源和行为审查。

## 修复策略

- 验证落实“来源证明、锁文件、脚本禁用策略、依赖审查和隔离构建”后的正常受控流程。
- 未知 `scenarioKey` 或 `controlKey` 必须阻断且不回显原始值。
- 事件日志只记录固定 key、风险标签、后端决策和学习信号。
- 正常受控流程必须通过“来源与策略已验证”验证，不得一刀切破坏正常业务。
