# CSRF 漏洞版

## 场景说明

漏洞版模拟一个只依赖登录态的转账接口：

```text
POST /api/labs/web/csrf/vuln/transfer
```

该接口要求用户已登录，但不要求请求体携带 CSRF token。因此在本机受控实验中，页面上的“模拟第三方请求”会被后端接受。

## 观察点

- 点击 `模拟第三方请求` 后，账户余额减少。
- 页面显示 `csrf-transfer-accepted` 对应的说明。
- 登录用户的验证记录会写入账户中心。

## 风险解释

如果真实业务只检查“用户是否登录”，而不确认请求是否来自当前可信页面，攻击者就可能诱导已登录用户的浏览器发起非本人意图的业务请求。

本项目不构造外部站点，不提供真实攻击投递能力，只用本机按钮模拟缺少 token 的请求。
