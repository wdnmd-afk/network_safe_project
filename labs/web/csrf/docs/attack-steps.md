# CSRF 漏洞版观察步骤

## 步骤

1. 启动本机前后端服务。
2. 使用 `demo_user / Demo@123456` 登录。
3. 打开 `/labs/web/csrf/vuln`。
4. 确认页面显示账户余额。
5. 点击 `模拟第三方请求`。
6. 观察账户余额减少，并确认页面信号为 `漏洞版接受了缺少 token 的请求`。

## 关键点

漏洞版的受控模拟请求仍然携带登录态，但请求体不携带 CSRF token。后端接受该请求，说明该业务只验证登录态，不验证请求是否来自可信页面。

## 边界

本步骤不需要也不允许构造外部站点。按钮只向本项目本机接口发送请求。
