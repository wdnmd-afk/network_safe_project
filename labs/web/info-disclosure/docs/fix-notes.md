# 信息泄露修复说明

## 根因

漏洞版把调试诊断报告暴露给普通用户。根因不是单个字段，而是调试能力和普通业务能力没有隔离。

## 本实验修复点

- 只允许普通用户读取公开报告。
- 对调试报告返回 403。
- 不返回堆栈、配置键名、token 形态和内部路径。
- 统一事件日志只记录摘要。

## 防御信号

```text
info-disclosure-debug-data-blocked
```

## 真实生产建议

真实生产中还应补充：

- 统一错误响应，不把异常堆栈返回给用户。
- 禁止在公网入口启用调试页面。
- 对内部诊断接口增加身份、角色和网络边界控制。
- 对日志、错误消息和响应体做敏感字段脱敏。
- 收敛 `Server`、`X-Powered-By`、版本号等响应头。
- 建立异常响应审计，追踪频繁探测调试信息的行为。

## 不在本实验中实现的内容

- 真实日志读取。
- 真实环境变量读取。
- 真实文件路径枚举。
- 真实 token 或密钥展示。
