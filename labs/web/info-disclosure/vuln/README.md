# 信息泄露漏洞版

## 漏洞点

漏洞版诊断接口只根据 `reportKey` 查找报告，没有把调试报告和普通用户隔离开。攻击方提交受控样例：

```text
debug-diagnostics
```

后端会返回内部调试诊断报告的教学占位内容，包括：

- 堆栈摘要
- 内部配置键名
- 演示 token 形态
- 内部路径占位

这些内容都不是项目真实敏感信息，只用于说明生产系统中调试响应暴露的风险。

## 观察重点

- 后端决策为 `accepted`。
- 学习信号为 `info-disclosure-debug-data-exposed`。
- 页面出现“调试报告”字段预览。
- 统一事件日志记录攻击阶段和报告摘要，但不保存完整调试内容。

## 风险理解

真实系统中的信息泄露通常不会直接完成攻击，但会帮助攻击者理解系统框架、路径、配置命名、认证形态和依赖版本，从而降低后续攻击成本。
