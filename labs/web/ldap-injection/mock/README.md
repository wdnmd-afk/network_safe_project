# LDAP 注入 mock 说明

## 目标

本目录只保存 LDAP 注入案例化实验的脱敏伪案例说明，不保存真实目录服务数据。

## 允许的伪数据

后续案例文档可以使用以下固定标签：

- `member-search-scope-risk`：组织成员搜索范围风险。
- `group-visibility-boundary-risk`：权限组可见范围风险。
- `login-candidate-filter-risk`：登录候选用户定位风险。

允许使用的伪角色：

- `普通成员`
- `组织管理员`
- `审计员`

允许使用的伪组织：

- `演示组织 A`
- `演示组织 B`

## 禁止内容

- 不保存真实 DN。
- 不保存真实邮箱、手机号或用户名。
- 不保存真实目录账号或凭据。
- 不保存真实组织结构。
- 不保存可连接的 LDAP 地址。
- 不保存可复制的过滤器 payload。

## 使用约束

前端工作台和虚拟目录 API 只能使用这些固定案例标签、伪角色说明和内存虚拟目录数据，不得从外部目录服务加载数据。
