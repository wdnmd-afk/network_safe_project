# LDAP 注入攻击方观察步骤

## 适用边界

本文只用于本项目的案例化学习，不是对外 LDAP 攻击指南。

本文不提供：

- 可复制过滤器 payload。
- LDAP 连接地址。
- 目录账号或凭据。
- 对外查询脚本。

## 1. 识别输入点

攻击方首先观察哪些输入可能进入目录查询流程：

- 组织成员搜索中的姓名关键词。
- 权限组查询中的组名关键词。
- 登录筛选中的用户名。

观察目标不是立即尝试攻击字符串，而是判断输入是否可能影响查询结构。

## 2. 判断服务端是否固定业务边界

继续观察业务边界来自哪里：

- 当前组织范围是否由服务端上下文固定。
- 权限组可见范围是否由服务端权限判断固定。
- 登录候选用户定位是否有固定流程。

如果这些边界来自客户端输入或可拼接条件，风险就会升高。

## 3. 观察漏洞案例中的语义偏移

三个固定案例的观察点如下：

| 案例 | 正常意图 | 风险意图 |
|---|---|---|
| 组织成员搜索 | 在当前组织内按姓名查找成员 | 输入影响组织范围后，结果范围偏离当前组织 |
| 权限组查询 | 在允许范围内按组名查找 | 输入影响权限边界后，可能看到不该展示的组 |
| 登录筛选 | 按用户名定位候选用户并继续凭据校验 | 输入影响候选定位后，登录流程出现异常风险 |

## 4. 使用前端工作台观察受控样例

当前页面已接入受控虚拟目录 API。攻击方视角的观察方式是：

1. 打开 `/labs/web/ldap-injection/vuln`。
2. 选择固定目录场景，例如 `member-search`。
3. 先提交正常关键词，观察 `ldap-injection-safe-search-completed`。
4. 点击页面提供的受控样例按钮，再提交请求。
5. 观察漏洞版是否返回 `ldap-injection-scope-expanded`，以及结果范围是否变为 `expanded`。

该受控样例只是本项目固定教学标记，不是可用于真实目录服务的 LDAP 过滤器。

## 5. 观察日志与复盘重点

当前阶段已有受控虚拟目录 API，但仍不连接真实目录服务。学习者应在页面、文档和服务端测试中复盘：

- 输入点在哪里。
- 服务端边界是否固定。
- 漏洞意图如何改变查询语义。
- 修复策略如何保证输入只作为文本值。
- 事件日志是否只记录场景、长度、脱敏预览、风险类型、结果范围和学习信号。

服务端测试覆盖的学习信号包括：

- `ldap-injection-safe-search-completed`
- `ldap-injection-scope-expanded`
- `ldap-injection-controlled-sample-blocked`

## 6. 禁止操作

- 不连接真实 LDAP / AD / OpenLDAP 服务。
- 不请求外部目标。
- 不读取本机目录配置。
- 不构造或保存可复制过滤器 payload。
- 不保存真实账号、DN、邮箱、手机号或凭据。
