# LDAP 注入修复案例

## 目标

修复案例用于说明：LDAP 注入防护必须在服务端完成，不能依赖前端隐藏输入框或提示用户不要输入特殊字符。

## 核心修复点

### 服务端查询模板

目录查询结构应由服务端固定：

- 当前组织范围由服务端上下文决定。
- 当前用户可见权限组由服务端权限判断决定。
- 登录候选用户定位只接受固定字段和固定流程。

用户输入只能作为普通文本值进入模板，不能改变查询结构。

### 字段允许列表

允许被搜索的字段必须固定，例如：

- 成员显示名。
- 成员工号的脱敏教学字段。
- 权限组展示名的伪字段。

客户端不能提交任意目录字段名，也不能提交权限范围条件。

### 过滤器转义

进入目录过滤条件的文本值必须做上下文转义。转义的目标是保持“文本值就是文本值”，避免它变成过滤条件结构的一部分。

本项目当前只提供受控虚拟目录 API，不实现真实过滤器转义函数。该 API 用固定场景展示服务端模板化与阻断差异，不连接真实目录服务。

### 最小权限与审计

真实生产还需要：

- 目录查询账号最小权限。
- 登录失败计数和账号策略。
- 高风险查询审计。
- 异常结果范围告警。
- 服务端统一脱敏日志。

## 日志边界

当前前端工作台提交到受控虚拟目录 API 后，也只能记录：

- 场景 key。
- 关键词长度和脱敏预览。
- 风险标签。
- 结果范围。
- 学习信号。
- 阻断或放行决策。

不得记录完整危险输入、LDAP 过滤器字符串、目录账号、DN、邮箱、手机号、凭据或外部目标信息。
