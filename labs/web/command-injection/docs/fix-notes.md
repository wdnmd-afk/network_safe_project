# 命令注入修复说明

## 漏洞根因

命令注入的根因是系统把用户输入拼进可解释的命令流程中，导致输入不再只是数据，而可能改变执行语义。

本项目中的漏洞版只做虚拟模拟，不执行真实命令。

## 修复策略

修复版采用以下策略：

- 只接受固定诊断任务 ID。
- 任务 ID 必须来自允许列表。
- 用户输入只作为普通参数值。
- 检测到命令连接符、管道或重定向时阻断。
- 日志只记录风险类型和学习信号，不记录完整输入。

## 预期防御信号

提交受控攻击样例时，修复版应返回：

```text
command-injection-allowlist-blocked
```

正常目标仍应返回：

```text
command-injection-normal-task-completed
```

## 不做的事情

- 不调用真实 shell。
- 不调用 PowerShell 或 `cmd.exe`。
- 不调用 `child_process`。
- 不执行用户输入。
- 不访问外部目标。
