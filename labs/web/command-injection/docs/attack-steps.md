# 命令注入攻击步骤

## 前置条件

- 已登录平台演示账号。
- 后端服务运行在本机。
- 只使用本项目页面或本机脚本。

## 攻击方视角

1. 进入 `/labs/web/command-injection/vuln`。
2. 保持诊断任务为 `cache-status`。
3. 点击“填入攻击样例”。
4. 确认虚拟目标为 `storefront-cache && reveal-debug-note`。
5. 点击“运行诊断”。
6. 观察后端决策为 `accepted`。
7. 观察学习信号为 `command-injection-virtual-command-executed`。
8. 观察虚拟步骤列表中出现“额外虚拟调试信息”。

## 复盘问题

- 攻击者控制了哪个输入？
- 为什么命令连接符会改变后端解释方式？
- 漏洞版为什么返回了虚拟额外步骤？
- 如果切换到修复版，同样输入应该被哪个边界阻断？

## 边界说明

本实验不会执行真实系统命令。攻击样例只用于触发虚拟命令运行器中的学习信号。
