# XPath 注入漏洞版规划

## 目标

漏洞版用于展示“用户输入影响 XML 查询条件语义”后，虚拟产品目录结果范围可能被扩大。

## 计划行为

- 后端使用内存中的虚拟 XML 产品目录数据集合。
- 正常关键词只返回公开产品目录中的匹配结果。
- 固定受控样例 `LAB_CONTROLLED_XPATH:expand-product-catalog` 触发教学模拟信号。
- 结果范围扩大只发生在虚拟查询模拟器中，不执行真实 XPath 表达式。
- 事件日志只记录关键词长度、脱敏预览、风险类别、结果范围和学习信号。

## 禁止行为

- 不读取真实 XML 文件。
- 不执行任意 XPath 表达式。
- 不连接外部 XML 数据源。
- 不访问外部目标。
- 不保存完整危险输入或完整查询表达式。
- 不提供通用 payload 库。

## 计划观察点

- 正常关键词请求应出现 `xpath-injection-safe-query-completed`。
- 固定受控样例应出现 `xpath-injection-result-scope-expanded`。
- 日志应能说明该次请求是受控攻击模拟，但不能暴露完整输入。
