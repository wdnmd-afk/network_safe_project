# SSRF 修复说明

## 当前修复策略

修复版采用“白名单优先”的策略：

- URL 必须是合法格式。
- 协议只能是 `http:` 或 `https:`。
- 目标主机必须在公开白名单中。
- 私有地址、回环地址、链路本地地址和内部域名直接阻断。
- 实验仍只查询虚拟资源表，不发起真实网络请求。

## 为什么不能只校验 URL 字符串

- 攻击者可以使用 IP、内部域名、重定向、DNS 变化或编码形式绕过简单字符串判断。
- 仅检查是否以 `http` 开头不能证明目标是安全公开资源。
- 真实系统必须在解析后校验最终 IP，并对重定向后的每一跳重复校验。

## 真实生产还需要补充

- 出站代理统一控制可访问目标。
- 网络层阻断内网、元数据服务、管理网段和回环地址。
- DNS 解析后校验 IP 范围。
- 禁止或严格限制重定向。
- 限制响应大小、内容类型、超时和连接数。
- 对异常目标、失败次数和私有地址尝试进行审计告警。

## 当前实验刻意不做的事

- 不发起真实 HTTP 请求。
- 不访问真实云元数据服务。
- 不提供 SSRF 扫描能力。
- 不访问外部真实目标。
