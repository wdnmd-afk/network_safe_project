# 漏洞版说明

漏洞版模拟一个帮助中心文档预览接口：

1. 把允许根目录固定为 `public/`。
2. 把用户输入路径直接拼到 `public/` 后面。
3. 对拼接后的路径做规范化。
4. 没有检查规范化后的路径是否仍在 `public/` 内。

当攻击样例 `../internal/admin-note.txt` 被提交时，规范化结果变成 `internal/admin-note.txt`，漏洞版会从虚拟文档索引中返回内部模拟文档。

关键观察点：

- 输入中包含 `..`。
- 规范化后的路径逃逸了 `public/`。
- 统一事件日志会记录 `path-traversal-sensitive-file-exposed`。

本实现不读取真实文件系统，只用于观察路径拼接和根目录校验缺失带来的风险。
