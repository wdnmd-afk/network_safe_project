# 目录遍历修复说明

## 当前修复策略

修复版采用“先规范化，再校验根目录”的策略：

- 将 `\` 统一转换为 `/`。
- 去掉空路径段和 `.`。
- 处理 `..` 后得到最终路径。
- 确认最终路径仍位于 `public/` 下。
- 只要最终路径逃逸 `public/`，就返回 403。

## 为什么不能只做字符串包含判断

- 只判断输入是否包含 `..` 容易误伤合法路径，也可能漏掉编码、分隔符差异或多层规范化问题。
- 只检查前缀容易被 `public/../internal/file` 绕过。
- 必须基于规范化后的最终路径做根目录约束。

## 真实生产还需要补充

- 尽量使用文档 ID 或数据库记录映射文件，而不是让用户直接控制路径。
- 下载前检查当前用户是否有权限访问该文档。
- 使用对象存储或独立下载服务隔离真实路径。
- 错误响应不要暴露目录结构和内部文件名。
- 记录访问审计，识别频繁 `../` 尝试。
- 对路径长度、字符集和编码形式做统一约束。

## 当前实验刻意不做的事

- 不读取真实文件系统。
- 不演示操作系统敏感文件路径。
- 不提供通用任意文件读取工具。
- 不访问外部真实目标。
