# SSTI 修复说明

## 漏洞根因

服务端模板注入的根因是系统把用户可控内容当作模板源或模板表达式解释。攻击方一旦能控制模板语法，就可能触发表达式执行、上下文泄露或更严重的服务器端风险。

## 本实验修复策略

修复版采用以下策略：

- 模板来源由系统维护。
- 用户只提供变量值。
- 变量名固定为 `customerName`、`orderNo`、`noticeTitle`。
- 模板表达式只允许固定变量插值。
- 发现 `{{ 7 * 7 }}`、`{{ debugContext }}` 或未知表达式时直接阻断。
- 事件日志只记录脱敏摘要。

## 修复版预期行为

- 正常模板变量仍可渲染。
- 受控数学表达式返回 `ssti-expression-blocked`。
- 受控上下文样例返回 `ssti-expression-blocked`。
- 日志阶段为 `defense`，风险等级为 `medium`。

## 生产补充建议

真实生产中还需要：

- 选择安全模板引擎配置。
- 禁止用户控制模板源。
- 对模板变量做允许列表校验。
- 按 HTML、属性、URL、脚本等输出上下文做转义。
- 限制模板能力、运行资源和模板变更权限。
- 审计模板发布与变更流程。
