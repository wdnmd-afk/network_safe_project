# SSTI 攻击步骤

## 攻击方目标

攻击方尝试控制通知模板文本，让服务端把输入中的模板表达式当作语法解释，从而观察表达式执行或上下文暴露信号。

本实验只允许两个固定受控样例：

- `{{ 7 * 7 }}`
- `{{ debugContext }}`

它们只由教学模拟器识别，不会交给真实模板引擎或 JavaScript 执行。

## 前置条件

- 仅在本机平台运行。
- 已登录演示账号。
- 后端服务已启用统一实验事件日志。

## 漏洞版步骤

1. 打开 `/labs/web/ssti/vuln`。
2. 点击“填入正常模板”，提交一次预览，确认出现 `ssti-safe-template-rendered`。
3. 点击“填入表达式样例”，提交预览。
4. 观察后端返回 `ssti-expression-evaluated`。
5. 点击“填入上下文样例”，提交预览。
6. 观察后端返回 `ssti-template-context-exposed`。
7. 在账户中心或实验详情页查看统一事件日志，确认阶段为 `attack`，输入摘要不包含完整模板表达式。

## 攻击方观察点

- 表达式数量是否变化。
- 表达式类别是否出现 `controlled-math-expression` 或 `controlled-debug-context`。
- 漏洞版是否返回固定教学结果。
- 日志是否记录学习信号而不是完整模板全文。

## 禁止行为

- 不访问外部目标。
- 不生成通用 SSTI payload 库。
- 不读取本机真实文件。
- 不使用真实凭据、真实 token 或真实 Cookie。
- 不尝试执行真实模板表达式。
