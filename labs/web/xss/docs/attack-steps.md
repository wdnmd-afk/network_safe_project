# XSS 漏洞版观察步骤

## 目标

说明未转义输出如何让用户输入改变页面结构。本文档只面向本项目本机实验页面，不提供外部目标利用步骤。

## 样例 payload

```html
<mark data-xss-lab-signal="xss">XSS 模拟信号</mark>
```

该 payload 是惰性 HTML 标记，不包含 JavaScript，不读取 Cookie，不发送网络请求。

## 步骤

1. 启动本机前端与后端服务。
2. 访问 `/labs/web/xss/vuln`。
3. 点击“填入样例”。
4. 点击“提交留言”。
5. 观察留言预览区域。

## 预期现象

- 页面出现黄色 `XSS 模拟信号`。
- DOM 中存在 `data-xss-lab-signal="xss"`。
- 这说明用户输入没有被当作普通文本处理，而是进入了页面结构。

## 关键结论

漏洞不在于某一个固定字符串，而在于“不可信内容进入 HTML 上下文时缺少输出编码”。真实业务中应默认把用户内容作为文本处理。

## 安全边界

- 仅用于本机 `web/xss` 实验。
- 不对外部网站使用。
- 不扩展为 Cookie 读取、脚本执行或网络回连演示。
