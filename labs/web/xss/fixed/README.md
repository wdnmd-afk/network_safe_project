# XSS 修复版

## 目标

演示同样的客服留言输入在文本渲染下不会改变页面结构。

## 实现方式

修复版页面位于：

```text
/labs/web/xss/fixed
```

页面接收与漏洞版相同的输入，但使用文本渲染展示内容。HTML 标记会被原样显示，不会被浏览器解释为页面结构。

## 修复要点

- 用户输入默认按文本处理。
- 避免在不可信内容上使用 HTML 注入渲染。
- 后续如确需富文本，应先引入明确的白名单清洗策略。

## 验证要点

- 提交与漏洞版相同的样例 payload。
- 页面应原样显示 `<mark data-xss-lab-signal="xss">XSS 模拟信号</mark>`。
- DOM 中不应生成 `data-xss-lab-signal="xss"` 标记。
- 修复目标是让用户内容不能改变页面结构，而不是只拦截某一个固定字符串。

## 适用边界

当前修复版演示的是文本渲染这一最小策略。若后续业务确实需要富文本能力，应单独设计白名单标签、属性过滤、协议过滤和测试用例，不能直接回到 HTML 注入渲染。
