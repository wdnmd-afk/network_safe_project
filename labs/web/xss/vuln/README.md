# XSS 漏洞版

## 目标

演示客服留言内容被当作 HTML 输出时，用户输入可以改变页面结构。

## 风险说明

漏洞版模拟“未对用户内容做输出编码，直接按 HTML 注入页面”的问题。在真实系统中，这类问题可能造成脚本执行、页面钓鱼、会话信息暴露等风险。本项目只展示页面结构被改变这一层现象，不执行脚本。

## 实现方式

漏洞版页面位于：

```text
/labs/web/xss/vuln
```

页面使用本机固定样例：

```html
<mark data-xss-lab-signal="xss">XSS 模拟信号</mark>
```

提交后，页面会解释该 HTML 标记并显示黄色模拟信号。

## 观察点

- 留言预览中会生成带 `data-xss-lab-signal="xss"` 的 DOM 标记。
- 黄色 `XSS 模拟信号` 说明输入已经改变页面结构。
- 该行为来自漏洞版受控使用 HTML 渲染，后续场景不得把这种写法复制到正常业务页面。

## 安全边界

该样例不执行 JavaScript，不读取 Cookie，不发送网络请求，只用于说明未转义输出风险。
