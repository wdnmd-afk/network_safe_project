# NoSQL 注入修复版

## 目标

修复版展示服务端查询模板、字段允许列表和输入类型约束如何阻断 NoSQL 注入风险。

## 行为

- 只接受固定查询模式。
- 用户输入只作为普通字符串值。
- 拒绝对象型、数组型、疑似操作符型输入。
- 正常关键词仍可返回公开优惠券。
- 日志只记录脱敏摘要。

## 修复重点

- 查询结构由服务端维护。
- 客户端不能提交查询对象。
- 查询字段和比较方式都走允许列表。
- 风险输入在进入虚拟查询器前被阻断。

## 入口

页面：

```text
/labs/web/nosql-injection/fixed
```

API：

```text
POST /api/labs/web/nosql-injection/fixed/search
```
