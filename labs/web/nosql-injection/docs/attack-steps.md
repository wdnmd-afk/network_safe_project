# NoSQL 注入攻击步骤

## 攻击方目标

攻击者希望通过可控输入影响后端结构化查询条件，让本来不应该返回的虚拟优惠券文档出现在结果中。

## 前置条件

- 漏洞版页面已实现。
- 攻击者能访问本机实验页面。
- 攻击者能控制关键词和教学筛选文本。
- 后端漏洞版使用虚拟文档查询器解释受控样例。

## 操作步骤

1. 打开 `/labs/web/nosql-injection/vuln`。
2. 点击“填入正常查询”。
3. 提交查询，观察只返回公开优惠券。
4. 点击“填入受控样例”。
5. 提交查询，观察后端识别到结构化查询风险类别。
6. 观察漏洞版返回 `nosql-injection-query-expanded`。
7. 查看请求摘要，只应看到查询模式、长度、风险类别和学习信号。
8. 查看账户中心或实验详情页中的最近事件复盘。

## 成功信号

- 页面显示虚拟查询范围扩大。
- 后端返回 `nosql-injection-query-expanded`。
- 事件日志中 `phase = attack`、`decision = accepted`、`risk_level = high`。

## 刻意限制

本实验不会演示：

- 对真实 MongoDB 或其他 NoSQL 服务发起查询。
- 任意对象查询执行。
- 数据写入、删除或批量导出。
- 外部目标探测。
- 通用 NoSQL payload 库。
