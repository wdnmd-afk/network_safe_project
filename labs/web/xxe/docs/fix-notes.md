# XXE 修复说明

## 漏洞根因

XXE 的根因是 XML 解析器允许 DTD 或外部实体解析。攻击方一旦能控制 XML 文档，就可能诱导服务端读取本地资源、访问内部地址或暴露敏感配置。

## 本实验修复策略

修复版采用以下策略：

- 在解析前检测并阻断 DOCTYPE。
- 禁用外部实体声明。
- 禁止不可信实体引用。
- 限制 XML 文档长度。
- 只读取业务需要字段。
- 事件日志只记录脱敏摘要。

## 修复版预期行为

- 正常 XML 仍可导入。
- 受控 XXE 样例返回 `xxe-doctype-blocked`。
- 日志阶段为 `defense`，风险等级为 `medium`。
- 日志不保存完整 XML 文档或完整实体声明。

## 生产补充建议

真实生产中还需要：

- 使用安全 XML 解析器配置。
- 默认禁用 DTD 与外部实体。
- 限制 XML 大小、嵌套深度和解析耗时。
- 只读取业务需要字段。
- 对 XML 导入链路增加审计、告警和异常样本留存策略。
- 对上传和导入来源做权限与业务流程校验。
