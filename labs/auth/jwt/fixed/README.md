# JWT 攻击修复版

## 修复策略

修复版 JWT 验证器只允许 HS256 教学 token，并强制校验签名。遇到 `alg=none` token 时直接阻断。

修复版不会把 payload 里的角色声明单独当作授权依据。只有通过算法白名单和签名校验后，声明才进入后续判断。

## 观察重点

- 后端决策为 `blocked`。
- 学习信号为 `jwt-none-alg-blocked`。
- 页面显示算法为 `none`，签名不存在。
- 授权结果为 `not-granted`。
- 统一事件日志记录防御阶段和阻断原因。

## 生产补充

真实生产环境还应补充：

- 密钥轮换与密钥 ID 管理
- `iss` / `aud` / `exp` / `nbf` 校验
- token 撤销和会话失效策略
- 权限从服务端数据源读取，不只依赖 token claim
- 异常 token 频率审计与告警
