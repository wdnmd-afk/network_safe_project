# IDOR 修复版

## 修复策略

修复版仍允许用户按 `orderId` 查询订单，但后端必须在返回前校验对象归属：

```text
order.ownerUserId === currentUser.id
```

只要订单不属于当前登录用户，即使对象真实存在，也返回 `403`。

## 防御信号

同样提交：

```json
{
  "orderId": "order-2001"
}
```

修复版会阻断请求，学习信号为：

```text
idor-cross-user-order-blocked
```

## 真实生产补充

真实生产环境还需要：

- 服务端对象级授权统一封装。
- 避免只依赖前端隐藏按钮。
- 使用不可预测对象 ID 作为辅助措施，但不能替代授权。
- 记录越权尝试并建立告警。
- 最小化响应字段，避免错误响应泄露过多信息。
