# IDOR 攻击步骤

## 前置条件

- 已登录本项目本机演示账号。
- 后端服务运行在本机。
- 打开 `/labs/auth/idor/vuln`。

## 步骤

1. 点击“读取自己订单”，确认请求体为：

   ```json
   {
     "orderId": "order-1001"
   }
   ```

2. 点击“读取订单”，观察正常业务信号：

   ```text
   idor-own-order-accepted
   ```

3. 点击“替换为他人订单”，请求体变为：

   ```json
   {
     "orderId": "order-2001"
   }
   ```

4. 再次点击“读取订单”，观察漏洞版返回他人订单详情。

5. 后端日志和数据库事件日志应出现：

   ```text
   idor-cross-user-order-exposed
   ```

## 攻击方观察点

- 可控输入是 `orderId`。
- 漏洞根因不是 ID 可猜，而是后端缺少对象归属校验。
- 同一请求在修复版中应被阻断。

## 安全边界

本实验只提交固定受控样例，不做对象 ID 枚举、扫描或爆破。
