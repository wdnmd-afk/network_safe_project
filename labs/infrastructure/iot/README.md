# IoT 攻击

## 场景目标

通过固定设备清单学习默认身份、固件和网络分区风险。

本实验只使用固定案例 `synthetic-iot-device-exposure`，用于观察设备身份、固件更新和网络边界缺少统一治理，并对比漏洞版与修复版的后端判定、学习信号和统一事件日志。

## 前置条件

- 本机前端与后端服务已启动。
- 使用本项目本机演示账号登录。
- 不需要外部目标、第三方服务、真实凭据或真实业务数据。

## 使用方式

1. 访问 `/labs/infrastructure/iot/vuln`。
2. 使用固定案例和默认控制策略观察风险被接受。
3. 切换到 `/labs/infrastructure/iot/fixed`，观察高风险动作被阻断。
4. 选择“设备分区策略已验证”，确认正常受控流程仍可继续。
5. 在实验详情或账户中心复盘统一事件日志。

## 安全边界

- 只审阅固定虚拟配置和事件，不连接或修改真实基础设施。
- 页面和 API 只接受共享目录中声明的 scenarioKey 与 controlKey。
- 未知 key 会被脱敏阻断，不写入原始输入或外部目标信息。

该实验仅提供本机受控固定场景和只读验证，不允许扩展为通用攻击工具。
