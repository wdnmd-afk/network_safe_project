## 1. 目标

将 `nginx/` 从占位说明升级为可在 Windows 本机使用的配置模板和生成入口，实现：

- 前端 `apps/web/dist` 静态托管。
- Vue history fallback，深层实验路由刷新不返回 404。
- `/api/` 请求反向代理到本机 Node 服务 `6667`。
- 配置不写入个人绝对路径，不提交真实本机配置。
- 生成后可通过 nginx `-t` 做语法校验。
- 提供启动、停止、日志和故障排查说明。

## 2. 范围

- `nginx/network-safe.conf.template`。

- `nginx/README.md`。
- 代表性配置结构测试。
- 与 `LT-001` 发布验收文档的联动。

不修改机器上已有的其他 nginx 配置，不接管其他站点和端口。

## 3. 操作步骤

1. 读取仓库模板和本机 nginx 安装目录。
2. 接收前端 dist 路径、nginx 根目录、监听端口和 Node 端口。
3. 将 Windows 路径转换为 nginx 可识别的正斜杠路径。
4. 生成到临时或用户指定配置文件，不覆盖仓库模板。
5. 执行 `nginx -t` 语法验证。
6. 启动 nginx 前确认 Node 服务已经监听 `6667`。
7. 访问首页、深层实验路由和 `/api/health`。
8. 记录日志位置、停止命令和失败排查方法。

## 4. 实施建议

- 仓库只保存模板，不保存包含个人路径的生成配置。
- 默认监听 `8080`，避免占用常见生产端口。
- `proxy_pass` 保留 `/api` 原始路径，交给 Express 路由处理。
- 前端只由 nginx 提供静态文件，实验逻辑继续由 Node 服务负责。
- 生成脚本默认执行路径和文件存在性校验。
- 可通过 `MYSQL_CLI_PATH` 类似方式提供 nginx 可执行文件的显式路径，但不把本机路径写入代码。

## 5. 潜在风险分析

| 风险 | 影响 | 控制方式 |
|---|---|---|
| dist 路径不存在 | nginx 启动后返回 404 | 生成前强制检查目录和 `index.html` |
| 模板变量未替换 | nginx 配置语法或运行错误 | 生成后扫描未替换占位符 |
| 代理目标未启动 | `/api` 返回 502 | 启动前检查后端健康接口 |
| 覆盖其他 nginx 配置 | 影响用户已有站点 | 只生成独立配置并要求显式 `-OutputConfig` |
| Windows 路径反斜杠 | nginx 解析失败 | 统一转换为正斜杠并加引号 |
| 生成配置含个人路径 | 不应提交到 Git | 默认输出到临时目录，`.gitignore` 排除本地配置 |
