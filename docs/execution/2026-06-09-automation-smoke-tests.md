# 自动化冒烟测试执行文档

## 1. 目标

补齐项目当前阶段的最小自动化冒烟测试能力，验证以下链路：

- 后端健康检查接口可访问
- 前端首页可访问
- 前端 `/api/*` 代理可以转发到后端
- 本机开发端口约定 `6670` / `6667` 与测试工具一致

本阶段不实现 Playwright 浏览器端到端测试，不验证具体漏洞触发行为。

## 2. 范围

本次修改范围：

- `packages/testing`
  - 冒烟测试配置
  - 冒烟测试运行器
  - 配置单元测试
  - README
- 根目录 `package.json`
  - 新增 `test:smoke`
  - 新增 `test:automation`
- `apps/web/vite.config.ts`
  - 增加 `strictPort`，保证冒烟测试端口确定
- `docs/testing/smoke-tests.md`
  - 记录运行方式与覆盖范围
- `docs/TODO.md`
  - 同步自动化测试进度

本次不修改：

- 数据库 schema
- 实验场景业务代码
- 漏洞版 / 修复版实现
- nginx 配置

## 3. 操作步骤

1. 在 `packages/testing` 新建包配置与测试入口。
2. 先写冒烟配置测试，验证：
   - 前端地址为 `http://127.0.0.1:6670`
   - 后端地址为 `http://127.0.0.1:6667`
   - 至少包含首页、后端健康检查、前端代理健康检查三个检查项
3. 实现冒烟配置与运行器：
   - 优先复用已运行服务
   - 如服务未运行，则启动本机前后端服务
   - 只清理本次运行器自己启动的进程
4. 在根脚本接入：
   - `pnpm test:smoke`
   - `pnpm test:automation`
5. 同步测试文档与 TODO。
6. 运行最小验证。

## 4. 实施建议

- 冒烟测试使用 Node 原生能力和 `fetch`，避免一开始引入 Playwright 浏览器安装成本。
- 启动前端时直接调用本地 Vite CLI，避免 Windows 下 `pnpm.cmd` 后台进程保持不稳定。
- 后端启动时显式设置 `PORT=6667` 与 `WEB_ORIGIN=http://localhost:6670`，避免本机 `.env` 旧值影响测试。
- HTTP 检查以状态码和少量响应内容断言为主，不做 UI 细节断言。

## 5. 潜在风险分析

- 如果 `6670` 或 `6667` 被无关进程占用，冒烟测试会失败并提示具体检查项。
- 如果已有服务占用端口但不是本项目服务，HTTP 内容断言会阻止误判通过。
- 当前冒烟测试不覆盖真实浏览器渲染，不等价于 Playwright E2E。

## 6. 优化方案

后续可在本基础上扩展：

1. Playwright 首页与实验入口端到端测试
2. `web/xss` 漏洞版 / 修复版对照测试
3. 实验脚本验证统一入口
4. build 后 nginx 场景冒烟测试

## 7. 验证方式

```powershell
pnpm test:automation
```

该命令应顺序执行：

- 后端测试
- 前端测试
- 共享测试占位
- 冒烟测试
