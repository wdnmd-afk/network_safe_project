# 前端开发端口调整到 6670 执行文档

## 1. 目标

将本机前端开发端口从 `6666` 调整为 `6670`，避免浏览器对受限端口的访问拦截；后端开发端口继续保持 `6667`。

## 2. 范围

本次修改范围：

- `apps/web` 前端运行配置、README 和对应测试
- `packages/testing` 冒烟测试与 Playwright 配置
- `apps/server/.env.example` 中的前端来源示例
- `docs/testing/` 与相关执行文档中的端口说明
- `labs/web/xss/docs/manual-verification.md` 中的手动访问地址

本次不修改：

- 后端 API 端口与接口行为
- 数据库 schema
- XSS 实验业务逻辑
- nginx 配置
- 任何真实 `.env` 文件

## 3. 操作步骤

1. 将前端统一运行端口配置改为 `6670`。
2. 同步冒烟测试和 Playwright 的前端基准地址。
3. 移除 Playwright 针对旧受限端口的特殊启动参数。
4. 同步单元测试断言和示例 URL。
5. 更新文档中的本机访问地址与端口说明。
6. 执行最小必要验证：
   - 前端配置测试
   - testing 包配置与运行器测试
   - 必要时启动前端开发服务确认 `6670` 可访问

## 4. 实施建议

- 后端端口继续使用 `6667`，避免扩大改动范围。
- 前端端口继续通过 `apps/web/src/config/runtime.ts` 单点配置，保持 Vite 配置和测试口径一致。
- 自动化测试统一使用 `127.0.0.1:6670`，浏览器手动访问文档统一使用 `localhost:6670`。

## 5. 潜在风险分析

- 本机 `6670` 若被其他进程占用，Vite 因 `strictPort` 会启动失败，需要先释放端口或再调整约定。
- 旧的 `6666` 前端服务可能仍在后台运行，用户需要访问新的 `6670` 地址。
- 若真实 `.env` 中仍保留旧 `WEB_ORIGIN`，后端跨域配置可能与新前端地址不一致；本次只更新示例文件，不读取或提交真实 `.env`。

## 6. 安全边界

本次只调整本项目本机开发端口，不新增攻击模拟能力，不访问外部目标，不修改漏洞实验的执行逻辑。

## 7. 验证方式

```powershell
pnpm.cmd --filter @network-safe/web test -- --run
pnpm.cmd --filter @network-safe/testing test
```

如需确认浏览器访问：

```powershell
pnpm dev:web
```

然后访问：

```text
http://localhost:6670/
```
