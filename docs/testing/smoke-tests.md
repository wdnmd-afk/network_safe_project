# 冒烟测试说明

## 1. 目标

冒烟测试用于快速确认当前本机开发链路是否可用，不替代单元测试、API 集成测试或 Playwright 端到端测试。

当前覆盖：

- `GET http://127.0.0.1:6666/`
- `GET http://127.0.0.1:6667/api/health`
- `GET http://127.0.0.1:6667/api/labs`
- `GET http://127.0.0.1:6666/api/health`

第三项用于验证后端能扫描并返回实验元数据列表；第四项用于验证前端 Vite 开发代理能将 `/api/*` 转发到后端服务。

## 2. 运行命令

```powershell
pnpm test:smoke
```

完整自动化入口：

```powershell
pnpm test:automation
```

## 3. 服务启动策略

冒烟测试运行器会先检查目标服务是否已经可用：

- 已可用：直接复用现有本机服务
- 不可用：启动本次测试需要的前端或后端服务

运行器只会清理自己启动的进程，不会关闭用户已有的服务进程。

## 4. 实现说明

冒烟测试使用 Node `http` / `https` 模块发起请求，不使用内置 `fetch`。

原因是当前前端端口固定为 `6666`，该端口在部分 Fetch 实现中属于受限端口，直接使用 `fetch` 会被运行时拦截，导致本机服务实际可访问但测试误报失败。

## 5. 当前限制

- 当前冒烟测试是 HTTP 级检查，不打开真实浏览器。
- 当前不验证漏洞版 / 修复版差异。
- 当前不依赖数据库探活，避免 MySQL 本机状态影响基础联通验证。

## 6. 后续扩展

后续建议增加：

- `web/xss` 漏洞版 / 修复版对照测试
- build 后 nginx 托管场景冒烟测试
- 实验脚本验证统一入口
