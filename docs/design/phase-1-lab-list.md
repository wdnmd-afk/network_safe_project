# 一期实验清单

## 1. 文档目标

本文件用于定义项目第一阶段要优先落地的实验内容。

它解决四个问题：

1. 一期到底先做哪些实验
2. 每个实验属于什么类别
3. 每个实验是否提供漏洞版与修复版
4. 每个实验后续应落到哪个目录

## 2. 一期范围原则

一期不追求覆盖你提供的全部攻击全景图，而是优先建设最适合做成平台核心能力的内容。

一期选择原则如下：

- 优先选择能通过前后端交互直接观察效果的内容
- 优先选择能体现漏洞版 / 修复版对照价值的内容
- 优先选择后续自动化测试容易接入的内容
- 优先选择能为后续网络实验、社会工程学、AI 模块提供平台基础能力的内容

## 3. 一期目标

一期目标不是把所有漏洞做完，而是先把平台主干跑通：

- 跑通 Vue 前端学习入口
- 跑通 Node 平台核心服务
- 跑通 MySQL 数据存储
- 跑通实验目录、学习记录、验证记录
- 跑通漏洞版 / 修复版对照模式
- 跑通脚本目录与验证入口

## 5. 一期优先级说明

### 高优先级

这些实验应优先进入第一轮实现：

- XSS
- CSRF
- 文件上传漏洞
- 目录遍历
- SQL 注入
- 命令注入
- 暴力破解
- 会话固定
- JWT 攻击
- 权限提升
- IDOR

原因：

- 学习价值高
- 可直接展示漏洞触发与修复差异
- 前后端与数据库协作价值明显
- 自动化验证较容易设计

### 中优先级

这些实验仍属于一期范围，但可在高优先级稳定后进入：

- SSRF
- 信息泄露
- SSTI
- XXE

原因：

- 价值仍然高
- 但部分内容依赖额外脚本、额外模拟资源或更明确的实验边界

## 6. 每个实验至少要包含什么

一期中的每个实验至少要具备以下内容：

1. 实验标题
2. 实验分类
3. 风险说明
4. 攻击目标
5. 漏洞版入口
6. 修复版入口
7. 攻击步骤
8. 修复说明
9. 验证方式
10. 是否支持脚本验证
11. 对应学习记录
12. 对应验证记录

## 8. 一期不包含什么

以下内容虽然属于总范围，但不作为一期实现重点：

- DDoS
- BGP 劫持
- 钓鱼与社会工程学模块
- 恶意软件模块
- 供应链模块
- AI / 新型攻击模块
- 客户端攻击模块
- 基础设施高级场景

这些内容应在二期及之后逐步接入。

## 9. 一期完成的判定标准

当以下条件满足时，可以认为一期实验层面达到可继续扩展的状态：

1. 高优先级实验目录全部建立
2. 每个高优先级实验都具备漏洞版与修复版
3. 平台可展示实验列表与实验详情
4. 学习记录与验证记录能跑通
5. 至少一部分实验具备脚本验证入口
6. 至少一部分实验具备自动化测试入口

## 10. 当前结论

一期不是“做几个最简单的 demo”，而是建立平台最关键的交互实验主干。

只要这一期的结构设计正确，后续网络层、社会工程学、AI、新型攻击和基础设施内容都可以按同一平台继续扩展。
