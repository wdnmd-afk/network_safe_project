# 项目范围与安全内容清单

## 1. 项目到底要做什么

本项目要建设的是一个运行在 Windows 本机上的网络安全学习平台，而不是单纯的漏洞代码集合。

平台的核心职责包括：

1. 给出统一的学习入口与实验目录
2. 将常见安全问题按类别组织成可学习的模块
3. 在适合的场景中同时提供漏洞版与修复版
4. 为不适合真实靶场化的内容提供模拟实验、案例演示或脚本实验
5. 记录学习进度、验证结果和后续自动化测试状态

## 2. 项目不做什么

为了保持边界清晰，本项目不以以下目标为导向：

- 不做对外真实目标的攻击工具集合
- 不做渗透测试接单工具箱
- 不追求把所有攻击都做成一个网页点击即可复现
- 不为了覆盖面牺牲结构清晰度和学习价值

## 3. 内容覆盖原则

你提供的攻击方式全景图会作为平台总范围，但不会全部用同一种方式实现。

本项目对安全内容采用三种落地方式：

### 3.1 真实交互靶场

适用于可以直接通过 Web 前后端、数据库和业务逻辑展示的问题。

### 3.2 脚本实验或本机模拟

适用于网络层、基础设施、部分客户端与辅助攻击链内容。

### 3.3 案例化与半交互演示

适用于社会工程学、AI 新型攻击、部分恶意软件与供应链内容。

## 4. 安全内容总清单

以下清单不是简单展示，而是后续要被纳入平台规划与阶段拆分的总范围。

### 4.1 Web 层

- XSS
- CSRF
- 点击劫持
- SSRF
- 开放重定向
- 文件上传漏洞
- 目录遍历
- 信息泄露

#### 建议落地方式

- 以真实交互靶场为主
- 尽量提供漏洞版与修复版
- 适合作为平台一期重点内容

### 4.2 注入类

- SQL 注入
- NoSQL 注入
- 命令注入
- LDAP 注入
- XPath 注入
- CRLF 注入
- 模板注入（SSTI）
- XXE 注入

#### 建议落地方式

- SQL 注入、命令注入、SSTI、XXE 优先做真实交互实验
- 其余内容可根据实现复杂度决定是交互实验还是案例化说明

### 4.3 认证 / 授权

- 暴力破解
- 凭据填充
- 会话劫持
- 会话固定
- JWT 攻击
- OAuth 漏洞
- 权限提升
- IDOR

#### 建议落地方式

- 以真实交互靶场为主
- 与业务逻辑漏洞一起构成平台一期核心内容

### 4.4 网络 / 传输层

- DDoS
- 中间人攻击
- DNS 劫持 / 污染
- ARP 欺骗
- 窃听攻击
- DNS 隧道
- 端口扫描
- BGP 劫持

#### 建议落地方式

- 端口扫描、DNS 隧道、基础 MITM 演示可做脚本实验或本机模拟
- DDoS、BGP 劫持等不适合做真实本机靶场，宜做原理模拟与案例讲解

### 4.5 社会工程学

- 网络钓鱼
- 鱼叉式钓鱼
- 捕鲸攻击
- 短信钓鱼
- 商业邮件诈骗
- 水坑攻击
- 钓鱼 WiFi
- 尾随 / 物理入侵

#### 建议落地方式

- 以案例、仿真页面、邮件模板、题目判断和流程演练为主
- 不作为真实攻击工具功能开发

### 4.6 恶意软件

- 勒索软件
- 木马
- 蠕虫
- 间谍软件
- 键盘记录器
- Rootkit

#### 建议落地方式

- 以样本分析、行为说明、演示流程和安全化模拟为主
- 不做真实破坏性样本运行

### 4.7 供应链

- 依赖混淆
- 恶意包注入
- 更新投毒
- 硬件供应链

#### 建议落地方式

- 以前端/后端依赖案例、伪包演示、更新链路说明和案例分析为主

### 4.8 AI / 新型攻击

- AI 驱动攻击
- Deepfake
- 对抗性 AI
- 加密劫持
- Prompt 注入

#### 建议落地方式

- Prompt 注入优先做可交互演示
- 其余内容以案例说明、演示流程和题目化学习为主

### 4.9 客户端攻击

- 驾车式下载
- 恶意浏览器插件
- Formjacking
- 恶意广告
- 浏览器 MITB

#### 建议落地方式

- 以仿真页面、脚本演示、浏览器行为说明和案例分析为主

### 4.10 基础设施

- 配置错误利用
- 容器逃逸
- 云安全漏洞
- IoT 攻击
- 零日漏洞利用

#### 建议落地方式

- 配置错误利用优先做本机模拟
- 容器逃逸、零日利用等仅做边界说明、案例化展示和学习材料，不做真实危险复现

## 5. 项目实施分期

### 第一期：平台骨架 + 核心交互靶场

重点范围：

- Web 层常见漏洞
- 注入类核心漏洞
- 认证 / 授权与业务逻辑问题
- 平台基础能力

目标：

- 跑通最小平台结构
- 跑通漏洞版 / 修复版对照模式
- 跑通学习记录和验证记录

### 第二期：网络实验与脚本体系

重点范围：

- 端口扫描
- DNS 相关实验
- 中间人攻击原理演示
- 配置错误利用
- 脚本目录与验证脚本体系

### 第三期：社会工程学、恶意软件、供应链与 AI 内容

重点范围：

- 钓鱼与邮件诈骗演示
- Prompt 注入
- 恶意软件行为演示
- 供应链案例
- AI 新型攻击学习模块

### 第四期：自动化测试体系完善

重点范围：

- 场景级验证脚本
- API 自动化测试
- 前端 E2E 测试
- 修复版回归验证

## 6. 平台中的每个实验应包含什么

后续每个实验尽量满足以下结构：

- 标题
- 分类
- 风险说明
- 攻击目标
- 漏洞版入口
- 修复版入口
- 攻击步骤
- 修复说明
- 验证方式
- 是否支持脚本实验
- 是否支持自动化验证

## 7. 当前结论

这个项目不是先写几个漏洞页面看看，而是要先定义清楚：

1. 平台有哪些安全内容
2. 每类内容如何落地
3. 哪些是一阶段重点
4. 哪些只能案例化或模拟化
5. 后续如何持续推进和记录进度

这些问题明确后，再开始搭工程骨架才是合理顺序。
