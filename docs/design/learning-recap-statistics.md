# 学习复盘统计说明

## 1. 文档定位

本文档说明账号中心“最近事件复盘”中的复盘完成度统计。

它用于帮助后续维护者和使用者理解：当前统计是基于最近事件的学习复盘提示，不是全量历史学习完成率，也不是攻击能力评分。

## 2. 统计目标

复盘完成度用于回答一个很具体的问题：

```text
在账号中心当前筛选范围内，最近事件对应的引导式复盘问题，当前用户完成了多少？
```

这个统计帮助学习者发现哪些最近操作还没有复盘，便于回到实验详情页逐项补齐问题。

## 3. 数据来源

当前统计只使用前端已经具备的两个数据源：

- 当前用户最近事件日志：`GET /api/lab-event-logs/me`
- 当前用户复盘问题完成记录：`GET /api/lab-recap-question-completions/me`

账号中心先按当前筛选条件读取事件日志，再提取事件中的 `traceId`，随后读取这些事件对应的复盘问题完成记录。

当前不新增后端统计接口，也不从数据库直接做全量聚合。

## 4. 筛选范围

账号中心当前支持以下筛选条件：

- 实验：`labKey`
- 阶段：`phase`
- 风险：`riskLevel`

筛选条件会同时影响：

- 当前事件列表
- 复盘完成度分母
- 复盘完成度分子
- 按实验聚合后的统计项

因此，当用户切换实验、阶段或风险时，完成度百分比变化是预期行为。

## 5. 聚合维度

当前统计按 `labKey` 聚合。

每个聚合项包含：

- `labKey`：实验 key。
- `title`：事件中的实验标题。
- `eventCount`：当前筛选范围内该实验的事件数量。
- `completedQuestions`：当前筛选范围内已完成的问题数量。
- `totalQuestions`：当前筛选范围内的问题总数。
- `completionPercent`：四舍五入后的完成百分比。

展示文案应保留“最近事件”含义，避免被理解为实验整体完成度。

## 6. 计算规则

对每条当前事件执行以下流程：

1. 调用 `createLabEventRecapQuestions(event)` 生成该事件当前版本的固定引导式问题。
2. 将问题数量累加到对应实验的 `totalQuestions`。
3. 对问题序号从 `0` 到 `questions.length - 1` 逐项检查完成记录。
4. 只有存在 `traceId + questionIndex` 对应记录，且 `completed === true` 时，才累加 `completedQuestions`。
5. 对每个实验计算：

```text
completionPercent = round(completedQuestions / totalQuestions * 100)
```

如果当前事件列表为空，则返回空统计，不展示误导性的 0%。

## 7. 边界规则

### 7.1 不代表全量历史

当前统计不代表：

- 当前用户全部历史复盘完成度。
- 某个实验的整体学习完成率。
- 某个漏洞类型的掌握程度。
- 攻击成功率、防御成功率或安全能力评分。

原因是当前事件日志接口只面向最近事件列表，账号中心没有读取全量历史事件。

### 7.2 不统计越界问题

如果完成记录中的 `questionIndex` 超出当前问题数量范围，不计入完成数。

这样做可以避免旧问题数量、异常记录或未来问题调整影响当前统计。

### 7.3 不依赖问题文案

完成状态不使用问题文案作为 key。

当前完成状态使用：

```text
traceId + questionIndex
```

后续如需引入问题版本，应先设计 `question_version` 或等价字段，再调整统计口径。

## 8. 展示边界

账号中心复盘统计可以展示：

- 实验标题
- 最近事件数
- 已完成问题数
- 问题总数
- 完成百分比

账号中心复盘统计不展示：

- `inputSummaryJson`
- 真实密码
- 真实 token
- 真实 Cookie
- 真实密钥
- 完整攻击 payload
- 外部真实目标信息

复盘统计只做学习进度提示，不应鼓励对外攻击或通用攻击工具化。

## 9. 与详情页的关系

账号中心负责展示当前筛选范围内的统计概览。

实验详情页负责展示具体事件、引导式问题和逐项完成状态。

推荐学习流程：

1. 在账号中心查看最近事件复盘和完成度。
2. 发现某个实验还有未完成问题。
3. 进入该实验详情页。
4. 展开对应事件，逐项完成复盘问题。

## 10. 后续扩展约束

如果未来需要全量历史统计，应先补充新的设计文档，至少明确：

- 后端聚合接口。
- 统计时间范围。
- 分页或归档策略。
- 事件日志保留策略。
- 问题版本兼容策略。
- 是否需要按实验、分类、阶段或风险拆分。

在这些内容确认前，不应把当前最近事件统计改名为“总完成度”。

## 11. 验证清单

维护复盘统计相关逻辑时，应至少确认：

- 空事件列表返回空统计。
- 多实验事件按 `labKey` 聚合。
- 只统计 `completed: true`。
- 越界 `questionIndex` 不计入完成数。
- 筛选条件变化后会重新读取事件和完成记录。
- 页面不展示 `inputSummaryJson` 或敏感输入明细。
